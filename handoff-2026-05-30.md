## 2. What the extension is
The shipped English Coach extension, rebranded by display-name to "Say It Right". It has TWO halves:
- **Coach (existing, sidebar webview):** rewrite English → idiomatic + Chinese explanation; Translate; read-aloud TTS; clipboard watch; `@coach` chat participant; Review Deck.
- **Pronunciation player (NEW, Plan 1, command-opened editor-tab webview):** per-sentence prosody "stave" (stress ● / reduced · / nuclear word / **intonation arrow** / IPA) + model TTS + 0.25–4× speed + AB-repeat + Shadow loop + Export. Internal command ids `sayItRight.*`, titled **"Say It Right: Practice a Sentence"** / **"Say It Right: Analyze Selection"**.

## 6. CRITICAL Marketplace gotchas — do not relearn these (also in `memory/publishing.md`)
- **Unpublishing reserves BOTH `name` AND `displayName`** — cannot reuse either, even under a different publisher. The rebrand MUST use a new `name` (`vscode-say-it-right`) AND a new `displayName` ("…发音教练", since unpublishing English Coach locks "Say It Right 英语口语教练").
- **A brand-new listing that closely mirrors a live one → auto-rejected "suspicious content"** (duplicate heuristic). This already happened once with `vscode-say-it-right`. So **order matters: unpublish English Coach FIRST, then publish `vscode-say-it-right`.**
- **`VSCE_PAT` can publish to both publishers but could NOT unpublish under `XianweiZhang`** (different Azure DevOps org → TF400813). English Coach is under `Xianwei-Zhang` (the PAT's own publisher) — CLI unpublish *may* work but is untested; if it 403s, the user unpublishes via the web portal (Manage Publishers → extension → ⋮ → Remove).
- **Qwen/DashScope JSON mode requires the literal word "json" in the prompt** (fixed for prosody; keep in mind for any new structured call).
- **Webview cannot access the microphone** (`getUserMedia` blocked) — recording is Plan 3 via host-side ffmpeg (verified feasible: ffmpeg installed + VS Code declares mic entitlement, macOS-first).
- **`vsce package` + `ls -t *.vsix` picks a STALE vsix** — always `rm -f *.vsix` before packaging.
- **Webview assets must live under `media/` not `src/`** (`.vscodeignore` excludes `src/**` → assets would 404 in the packaged VSIX). They're at `media/player/`.
